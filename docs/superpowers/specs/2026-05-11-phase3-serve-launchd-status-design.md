# llamactl Phase 3: serve / launchd / status / stop — Design Spec

**Status:** Approved 2026-05-11 (this session).
**Builds on:** Phase 2.5 (merged at `5b222e6` on `main` 2026-05-11).
**Spec coverage:** PRD v1.5 §3 (`serve` / `stop` / `status` commands), §6.2 (recipe → flag mapping), §file-layout (LaunchAgents + Logs), §doctor (5 new checks). Targets PRD acceptance criteria **8, 9, 10, 13, 14, 15, 16**.

---

## 1. Goals

1. `serve <model-id>` runs llama-server in the foreground with hardware-tuned, recipe-driven flags. Ctrl-C cleanly stops the child.
2. `serve <model-id> --detach` registers a launchd LaunchAgent that auto-starts on user login and auto-restarts on crash. Returns within 5 seconds with a running PID.
3. `stop <model-id>` boots out the service and deletes the plist. `stop` (no arg) does this for all llamactl services.
4. `status [--json]` lists every llamactl-managed service with port, PID, memory, uptime, tokens/sec (rolling 60s window), and endpoint URL.
5. `doctor` gains 5 new checks: port conflicts, model files match metadata, orphaned metadata, disk space, Tailscale, stale plists.

Acceptance criteria addressed:

| AC | Requirement | Verified by |
|----|-------------|-------------|
| 8  | `serve <id> --detach` registers launchd service, returns within 5s, working endpoint at `http://localhost:8080` | §5.2; live smoke test |
| 9  | OpenAI chat-completions POST returns valid response | Live smoke test (llama-server is the OpenAI endpoint) |
| 10 | `status` shows running service; memory within 10% of `ps`/Activity Monitor; recent tok/s | §6.2; uses `ps -o rss=` directly |
| 13 | Clean reboot → service auto-starts; endpoint up within 60s | `RunAtLoad=true` in plist; live smoke test |
| 14 | Tailscale cross-host POST works | `--host 0.0.0.0` in flags; manual user verification |
| 15 | `stop <id>` stops cleanly within 10s; plist unloaded | §6.1 |
| 16 | Switching llamavm version takes effect on next `serve` | Plist regenerated each `serve --detach` |

## 2. Non-goals

- Multi-user / system-wide LaunchDaemons. We use per-user LaunchAgents only.
- Authentication on the endpoint. PRD §3 lists this as a non-goal; Tailscale handles access control.
- Speculative decoding auto-config. Out of scope per PRD.
- Hot model swapping. Each `serve --detach` is independent.
- Log rotation. Append-only; user/macOS handles cleanup if needed.
- Per-foreground-serve tracking. `status` only shows detached services. Foreground is interactive — its terminal is its UI.

## 4. Recipes (`internal/recipes`)

### 4.1 Data shapes

```go
type MlockMode int

const (
    MlockAuto MlockMode = iota // chat/code/long-context default
    MlockOff                   // low-memory always-off
)

type Recipe struct {
    Name       string
    CtxSize    int    // clamped to min(model.MaxCtx, this)
    CacheTypeK string // "f16", "q8_0", "q4_0"
    CacheTypeV string
    MlockMode  MlockMode
}

const DefaultRecipe = "chat"

var Recipes = map[string]Recipe{
    "chat":         {Name: "chat",         CtxSize: 8192,  CacheTypeK: "f16",  CacheTypeV: "f16",  MlockMode: MlockAuto},
    "code":         {Name: "code",         CtxSize: 16384, CacheTypeK: "f16",  CacheTypeV: "f16",  MlockMode: MlockAuto},
    "long-context": {Name: "long-context", CtxSize: 32768, CacheTypeK: "q8_0", CacheTypeV: "q8_0", MlockMode: MlockAuto},
    "low-memory":   {Name: "low-memory",   CtxSize: 4096,  CacheTypeK: "q4_0", CacheTypeV: "q4_0", MlockMode: MlockOff},
}

// MinFlashAttnBuild — empirical floor for stable --flash-attn support on
// Apple Silicon (upstream build ~2700, ca. 2024-08). llamavm-managed
// builds use cmake-counter Build values < 100; skip the floor for those.
const MinFlashAttnBuild = 2700
```

### 4.2 `FlagsFor` algorithm

```
Inputs:  recipe Recipe, m models.Model, q models.Quant, ggufPath string,
         hw hardware.Info, ver server.Version, sizeGB float64, port int

1. ctxSize = recipe.CtxSize; if m.MaxCtx > 0 && m.MaxCtx < ctxSize: ctxSize = m.MaxCtx
2. threads = max(1, platform.Cores() - 2)
3. Build base argv: --model, --host 0.0.0.0, --port, --ctx-size, --n-gpu-layers 999,
                    --cache-type-k, --cache-type-v, --threads
4. If recipe.MlockMode == MlockAuto and (GpuAddressableGB(hw) - OSOverheadGB - HeadroomGB) > sizeGB + 4:
       append --mlock
5. If shouldAddFlashAttn(ver):
       append --flash-attn
6. Return argv

shouldAddFlashAttn(ver) returns true iff ver.Build < 100 (cmake-counter custom build, assume modern)
                                       OR ver.Build >= MinFlashAttnBuild
```

## 5. launchd (`internal/launchd`)

### 5.1 Plist template

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>{{.Label}}</string>
  <key>ProgramArguments</key>
  <array>
    <string>{{.LlamaServer}}</string>
{{range .Args}}    <string>{{.}}</string>
{{end}}  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>WorkingDirectory</key>
  <string>{{.WorkingDir}}</string>
  <key>StandardOutPath</key>
  <string>{{.LogPath}}</string>
  <key>StandardErrorPath</key>
  <string>{{.LogPath}}</string>
  <key>ProcessType</key>
  <string>Interactive</string>
</dict>
</plist>
```

- `RunAtLoad=true` — start when user logs in.
- `KeepAlive=true` — restart on crash. Matches "service" semantics.
- `ProcessType=Interactive` — keeps macOS scheduler from de-prioritizing the process under load.
- Same `LogPath` for stdout and stderr → unified log file.

### 5.3 PID polling after Load

`serve --detach`'s post-condition is "service started within 5s". After `Load`, the controller polls `Print` every 250 ms for up to 5 s; returns the first `ServiceInfo` with `PID > 0`. After 5 s with no PID → `ErrUserError` with the log path so the user can investigate.

## 6. CLI commands

### 6.1 `serve <model-id> [--port <N>] [--recipe <name>] [--detach]`

```
1. metadata = ModelStore.Get(id). Missing → ErrUserError
   "model %q is not installed; run `llamactl add %s`"
2. hw = ensureHardware()    // Phase 2 helper
3. resolution = ServerResolver.Resolve()      // Phase 1 — must succeed
4. ver = ServerProber.Probe(resolution.Path)
   If ver.Build < MinLlamaServerBuild → ErrUserError
5. recipe = Recipes[--recipe or "chat"]; unknown → ErrUserError listing valid names
6. sizeGB = float64(metadata.SizeBytes) / (1<<30)
7. port = PortAllocator.Free(--port or 8080)
   If port != requested: Fprintf(stderr, "bound to :%d (:%d was in use)\n", port, requested)
8. argv = recipes.FlagsFor(recipe, model, metadata.Quant, metadata.GGUFPath,
                           hw, ver, sizeGB, port)
9. metadata.LastServedAt = Deps.Now(); ModelStore.Put(metadata)
10. If --detach: runDetached(); else: runForeground()

runForeground():
    mkdir -p ~/Library/Logs/llamactl
    logFile = OpenFile(~/Library/Logs/llamactl/<id>.log, O_APPEND|O_CREATE|O_WRONLY)
    cmd = exec.CommandContext(ctx, resolution.Path, argv...)
    cmd.Stdout = io.MultiWriter(logFile, d.Stdout)
    cmd.Stderr = io.MultiWriter(logFile, d.Stderr)
    Fprintf(d.Stdout, "starting llama-server (recipe=%s, port=%d)…\n",
            recipe.Name, port)
    return cmd.Run()    // ctx cancel (Ctrl-C from main.go) → SIGTERM to child

runDetached():
    label = "com.llamactl." + id
    plistPath = ~/Library/LaunchAgents/<label>.plist
    spec = launchd.PlistSpec{
        Label:       label,
        LlamaServer: resolution.Path,
        Args:        argv,
        LogPath:     ~/Library/Logs/llamactl/<id>.log,
        WorkingDir:  os.UserHomeDir(),
    }
    body, _ = launchd.Render(spec)
    mkdir -p ~/Library/LaunchAgents, ~/Library/Logs/llamactl
    atomicWrite(plistPath, body)   // tmp + rename

    // If an old service for this label is loaded, bootout first.
    if existing, _ := LaunchdService.Print(label); existing.PID != 0 {
        LaunchdService.Bootout(label)   // best-effort, log on failure
    }
    LaunchdService.Load(plistPath)

    // Poll for PID, up to 5s
    deadline = Now().Add(5*time.Second)
    for Now().Before(deadline):
        info, _ = LaunchdService.Print(label)
        if info.PID > 0:
            Fprintf(d.Stdout, "service started (pid=%d); endpoint http://localhost:%d\n",
                    info.PID, port)
            return nil
        sleep 250ms
    return ErrUserError "service didn't start within 5s; see ~/Library/Logs/llamactl/<id>.log"
```

### 6.2 `stop [<model-id>]`

```
1. agentsDir = ~/Library/LaunchAgents
2. If no arg:
     services = LaunchdService.List(agentsDir)
     if empty: Println("no llamactl services running"); return
     for each: stopOne(svc.Label, svc.PlistPath)
     return
3. With <id>:
     label = "com.llamactl." + id
     plistPath = filepath.Join(agentsDir, label+".plist")
     if !exists(plistPath): return ErrUserError "no detached service for %q"
     return stopOne(label, plistPath)

stopOne(label, plistPath):
    _ = LaunchdService.Bootout(label)    // best-effort; ignore "not bootstrapped" exit
    _ = os.Remove(plistPath)             // ignore ENOENT (some macOS versions auto-clean)
    Printf("stopped %s and removed %s\n", strings.TrimPrefix(label, "com.llamactl."), plistPath)
```

Foreground serves are not touched by `stop`. Their terminal is their UI.

### 6.3 `status [--json]`

```
1. services = LaunchdService.List(agentsDir)
2. if empty: Println("no detached services"); return
3. For each svc in services:
     info = LaunchdService.Print(svc.Label)
     port = parsePortFromPlist(svc.PlistPath)
     row = {ModelID, Port: port}
     if info.PID == 0:
         row.State = "stopped"
     else:
         row.PID = info.PID
         row.MemoryBytes, _ = ProcInspector.RSS(info.PID)
         row.UptimeSeconds, _ = ProcInspector.Uptime(info.PID)
         logPath = ~/Library/Logs/llamactl/<id>.log
         row.TokensPerSec, _ = TokRateReader.Rate(logPath, 60*time.Second, Now())
         row.Endpoint = fmt.Sprintf("http://localhost:%d", port)
         row.State = "running"

4. If --json:
     json.MarshalIndent(rows); Fprintln
5. Else:
     tabwriter columns: MODEL-ID  PORT  STATE  MEM  UPTIME  TOK/S  ENDPOINT
     "stopped" rows show "—" for MEM/UPTIME/TOK/S
     "running" rows with TokensPerSec == 0 show "—" for TOK/S (idle model)
```

JSON shape:

[

### 6.4 `doctor` — 5 new checks

Each runs after Phase 1's 4 checks. Same UI: ✓ / ✗ with cause + suggested fix.

1. **Port conflicts.** For each loaded service: try `net.Listen` on its declared port. If the bind succeeds, the service isn't actually holding the port (something's wrong). Loaded-but-bound is healthy.
2. **Model files match metadata.** Stat each `Metadata.GGUFPath`; check size within 1% of `Metadata.SizeBytes`. Don't recompute SHA.
3. **Orphaned metadata.** Inverse of (2): metadata exists but GGUF gone. Suggest `llamactl remove <id>`.
4. **Disk space available.** `syscall.Statfs(~/.local/share/llama-models)`. Fail if `< 5 GB`.
5. **Tailscale running (if configured).** Skip silently if `tailscale` not on PATH. Else `tailscale status --json` and parse `.Self.Online`. ✗ if false.
6. **Stale plists.** For each `com.llamactl.*.plist`: parse its `ProgramArguments[0]`; stat the path. If missing, report plist as stale; suggest `llamactl serve <id> --detach` to regenerate.

(That's 5 new + 1 stale-plist check = 6 additions. Counting the stale-plist as part of the launchd hygiene group; total `doctor` output is 10 lines.)

## 9. Error model

Same conventions as Phase 1+2:

- `ErrUserError` → exit 2:
  - model not installed (`serve`)
  - unknown recipe
  - llama-server below version floor
  - no free port in [preferred, preferred+100)
  - `stop <id>` with no matching plist
  - launchd didn't produce a PID within 5s after `Load`
- Generic (exit 1, `llamactl:` prefix):
  - `launchctl bootstrap` exec failure
  - plist write failure
  - log file open failure
  - `ps` failure during status
- Foreground `serve` exits with llama-server's own exit code so scripts can react to crashes.

## 11. Risks

- **`launchctl print` output format drift.** macOS 14 → 15 → 16 may tweak whitespace. We parse three key=value lines via prefix match (`pid =`, `state =`, `last exit code =`); resilient to surrounding text changes but brittle to key renames. Mitigation: if parsing returns zero PID despite Load succeeding, fall back to scanning `ps -axo pid,command | grep <llama-server-path>`. Defer that fallback until we see it bite.
- **PID staleness on launchd restart.** If `KeepAlive=true` and llama-server crashes, launchd restarts with a new PID. `status` always re-`Print`s, so the PID it shows is fresh. The `Inspector.RSS(pid)` call uses that fresh PID. No risk.
- **Two foreground serves of the same model in two terminals.** Both can load the same GGUF (read-only). They compete for ports — `PortAllocator.Free` finds different ones for each. Logs interleave in the shared log file. Acceptable: user opted into the chaos.
- **`status` while a serve is initializing.** `Print` returns PID=0 briefly (after Load, before launchd spawns). The 5s poll in `serve --detach` waits for PID>0; but a concurrent `status` call would show "stopped" for the same window. Tiny window, low risk.
- **Tok/s parser brittleness.** llama-server's per-request log line wording has changed minor revisions in 2023-2024. We match two known phrasings; if a future build changes again, tok/s silently returns 0. `status` still shows everything else correctly. Acceptable degradation.
- **Log file growth.** Append-only. A heavily-used model could generate megabytes per day. macOS log rotation doesn't auto-rotate `~/Library/Logs/`. Documented as a Phase 4+ concern; `doctor` could be extended later to warn on >100 MB.
- **Plist label collisions with non-llamactl services.** Any plist under `com.llamactl.*` is treated as ours. If a user manually creates `com.llamactl.foo.plist`, llamactl will try to manage it. Mitigation: name prefix is specific; no documented collision risk.
